5. 發現問題與修復記錄

問題 #1：投標管理倒數計時顯示異常

嚴重度中
位置BidPanel.jsx（Line 29-31）、Dashboard.jsx（Line 102-109）
現象Step 7 投標管理頁面的倒數計時器顯示為 77,134,426 天 或 73,260,306 天
影響Dashboard 專案卡片同樣出現天文數字的倒數天數
狀態已修復 ✓

根因分析

專案的 deadline 欄位存入了非法日期格式,如 "213213-02-21"（年份為 213213）。 JavaScript 的 new Date("213213-02-21") 不會拋出錯誤,而是解析為公元 213213 年, 導致計算出的天數差異超過 7,700 萬天。

修復前
77,134,426 天

deadline = "213213-02-21"

修復後
不顯示倒數

非法日期格式被防護攔截

修復方式

在 BidPanel.jsx 與 Dashboard.jsx 的日期計算邏輯前,新增日期格式驗證函式（isValidDate()）：

  • 使用正規表達式驗證 YYYY-MM-DD 格式
  • 限制年份範圍為 2000–2100
  • 使用 isNaN(parsed.getTime()) 排除無效日期（如 2026-02-30）
  • 非法日期時,BidPanel 不顯示倒數區塊,Dashboard 顯示「日期格式錯誤」灰色標籤

問題 #2：Console 偶現 API 404/400 錯誤

嚴重度低
位置瀏覽器 Console
現象分頁切換時偶見 404 /api/projects/[id]/analyze 和 400 /themes/direction-questions
影響不影響使用者操作流程,僅在 Console 中產生紅字
狀態低優先（前端靜默處理即可）

分析：404 /analyze 為前端在切換到 AI 分析頁時的預載請求,若該專案尚無分析結果則回傳 404； 400 /direction-questions 為前端在無有效文件的情況下觸發 AI 工坊,後端正確回傳驗證失敗。 兩者均不影響功能正確性,建議未來在前端加入靜默 catch 以避免 Console 紅字。
