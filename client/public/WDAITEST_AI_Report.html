1. 系統總覽與 AI 架構

1.1 AI 供應商與路由機制

WDAITEST 採用多供應商自動降級架構，支援以下 AI 引擎：

引擎模型用途優先順序費用
Google Geminigemini-2.5-flash文字 + 圖片生成 + 照片編輯1（最優先）免費額度大
Groqllama-4-scout-17b文字生成、快速問答2免費
Ollama（本地）qwen3:8b文字生成（離線可用）2（本地優先時為 1）免費
OpenAIgpt-4o / gpt-4o-mini高品質文字生成3付費
路由邏輯：系統啟動時偵測環境變數中的 API Key，優先使用 Gemini → Groq → OpenAI。任何引擎呼叫失敗時，自動降級到下一個可用引擎，確保零停機。

3.1 科室深度情境引擎

觸發邏輯

  • 自動觸發：建案時若填寫了「機關」或「公司」欄位，系統在回傳 201 後非同步呼叫 researchDepartment()，不阻塞使用者
  • 手動觸發：POST /api/projects/:id/dept-intelligence
  • 輸出：12 欄位結構化 JSON 情報卡，存入 dept_intelligence 表

3.3 主題包裝引擎（三階段）

Phase 1：方向選擇題生成

流程：讀取 PDF 文件 → 組合專案上下文 + 科室情報 → AI 產出 3 組方向選擇題（每題 4 選項）→ 前端顯示 → PM 選擇方向
PROMPT: direction-questions
你是台灣的資深${isTender ? '政府活動' : '企業活動'}企劃創意總監。 根據以下需求文件內容以及該單位的深度背景情報，產出 3 組「方向選擇題」 每個問題必須： - 問題本身要具體，必須緊扣這個科室的業務領域和核心議題 - 每題提供 4 個選項 - 選項要有差異性，代表不同的創意走向 - 選項的用詞要展現你對該領域的深度理解（使用專業術語、引用真實案例方向） ${contextInfo} ${deptIntelContext} ← 【注入科室情報】 嚴格以 JSON 陣列格式輸出： [{ "question": "...", "options": ["A", "B", "C", "D"] }] 以下是需求文件原文："""${docText}"""
